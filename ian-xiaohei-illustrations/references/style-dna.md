# 风格 DNA

## 一句话

INBILY 粉卫衣星钥少女进入小黑风白纸草图系统：纯白、极简、手绘、留白、克制、怪诞、产品草图感、中文手写感，结构清楚但不说明书。

## 必须

- 16:9 横版正文配图。
- 纯白背景：不要米色、暖灰、纸张纹理、渐变、阴影、噪点、复古纸感。
- 黑色手绘线稿为主：细线、轻微抖动、不机械、不矢量、不厚重描边。
- 大量留白：主体占画面约 40%-60%，至少 35% 空白，最好有一整块安静区域。
- 少量中文手写批注：最多 5-8 处，每处尽量 2-8 个字。
- 一张图只讲一个核心动作、结构、状态或隐喻。
- 结构要自然表达，不要在图上写结构类型名称。
- INBILY IP 必须作为动作主体参与画面，不要作为贴纸或角标。

## 颜色

- 黑色：主线稿、结构、框线、主要文字、主要物件。
- 粉色：只用于 INBILY IP 的卫衣、鞋、星形挂件和极少量品牌识别点。
- 红色：重点批注、问题、情绪点、关键提醒、结果。
- 橙色：主流程、路径、箭头、自动化流向、从 A 到 B 的移动关系。
- 蓝色：补充说明、脑内状态、系统状态、第二层解释、AI/assistant/自动化提示。

蓝色不是每张都必须用。颜色要克制，宁可少不要多。粉色要服务角色识别，不能压过黑白草图感。

## 角色画法

- 先画成黑色手绘线稿角色，再在卫衣、鞋、星形挂件上加克制粉色。
- 保留棕色短发、小呆毛、粉色 INBILY 卫衣、白短裤、星形挂件、粉白鞋。
- 表情温暖但克制，可以有一点认真、冷静、deadpan 的工作感。
- 角色必须正在执行一个动作：拉、扛、塞、捞、压、称、缝、剪、拧、守、推、接、拆、标记、回收。
- 不要让角色只是站在图边或对镜头微笑。

## 绝对不要

- 不要商业插画。
- 不要品牌宣传海报。
- 不要 PPT 信息图。
- 不要正式流程图。
- 不要课程课件。
- 不要可爱卡通海报。
- 不要儿童插画。
- 不要复杂架构图。
- 不要精致扁平插画。
- 不要科技感 UI。
- 不要真实 App 截图。
- 不要复杂背景、渐变、阴影、纹理。
- 不要把每个节点都解释清楚。
- 不要左上角写“Workflow 流程图 / 系统架构图 / 常见坑 / 路线图”等类型标题。
- 不要把 INBILY 参考图的标题栏、边框、粉色展板复制到正文配图中。
- 不要生成黑色小怪物、黑色吉祥物或无关角色。

## 审美方向

要怪诞、创意、有意思、简洁清爽、天马行空。

INBILY IP 可以温暖，但不能甜腻；可以可爱，但不能幼稚；可以品牌化，但不能像广告。
